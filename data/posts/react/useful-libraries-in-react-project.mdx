最近开发了自己的第一款独立产品，一个基于 Next.js 开发的 WebApp。项目中用到了一些非常好用的三方库，在这里做一个盘点，或许你的下一个项目也可以用到。


### shadcn/ui 组件库

- 官网地址：https://ui.shadcn.com
- 该组件库截至目前（2024-04-02）已经在 [Github](https://github.com/shadcn-ui/ui) 上收获了 54.6k 的 star

基于 [bestofjs](https://bestofjs.org/) 上的数据，应该是近年来前端最热门的组件库。 

// lib-1.png

我实际体验下来，感觉真的不错。我的感受是，它和之前的许多组件库，比如 ant-design、mantine 等等最大的区别在于高度的可定制化。传统的组件库一般是封装好以后暴露出 props 接口，使用者只能受限于这些暴露的 api，如果业务中有一些定制化的诉求，可能需要做一些不太优雅的 trick，甚至有些业务特定功能根本无法基于这些开源的组件库实现，这也是许多稍微有实力的公司喜欢自己做组件库的原因之一。

但 `shadcn/ui` 不一样，你可以直接通过 cli 工具下载它包含的某个组件到本地目录，然后在本地你可以对组件源码进行修改。它底层依赖的是 [radix](https://www.radix-ui.com/) 组件库，实际也没办法做到百分百可定制性，但确实要比其他组件库灵活太多了。并且，`shadcn/ui` 基于 `tailwind.css` 样式方案，调整组件样式非常方便。组件的默认样式、交互做得都挺精致的，用户体验很好。总之，它的开发体验和用户体验做得都不错，也难怪如此热门，建议还没尝试的同学下一个项目可以考虑用一用 `shadcn/ui`。

### react-string 动画库

- 官网地址：https://www.react-spring.dev/docs

这是 react 开发生态中最热门的动画库之一，也是 react 开发者想要实现复杂、优雅动画的最强武器之一。和大家熟悉的基于贝塞尔曲线的动画不一样，它是 spring 动画，通过 `mass`、`frication`、`tension`（质量、摩擦、张力）等参数来模拟现实的物理动画效果，动画要更逼真。下面的视频，是我产品中的一个功能，通过纸牌卡片的方式复习记录的生词，就是用 `react-spring` 实现的，效果如下：

### react-hotkeys-hook 绑定快捷键的 hook

- 官网地址：https://react-hotkeys-hook.vercel.app
- 该项目截至目前已经在 [Github](https://github.com/JohannesKlauss/react-hotkeys-hook) 上收获了 2.4k 的 star

如果产品中包含了很多的交互，如果像提高用户体验，提供一些必要的快捷键是非常好的选择。我们可以自己在 useEffect 里去绑定一些快捷键事件，但类似这样的基础功能确实不太有必要自己造轮子，不需要自己考虑各种边界和性能问题，使稳定成熟好用的三方库可能是更好的选择。`react-hotkeys-hook` 就是这样一个库，使用很简单，示例代码如下：

```jsx
import { useHotkeys } from 'react-hotkeys-hook'

export const ExampleComponent = () => {
  const [count, setCount] = useState(0)
  useHotkeys('ctrl+k', () => setCount(count + 1), [count])

  return (
    <p>
      Pressed {count} times.
    </p>
  )
}
```
